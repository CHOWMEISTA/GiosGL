GiosGL

An advanced, dependency-free JavaScript library for ultra-high-resolution GPU rendering with WebGL2.

Get Started

Engine Features

4K Native Architecture

Automatic 4K initialization with smart resizing and aspect preservation. Bypasses browser upscaling for crisp, pixel-perfect rendering.

Low Latency Pipeline

Configured with desynchronized: true and discrete GPU power preferences to minimize input-to-display latency.

Resource Management

Built-in registries for Shaders, Textures, VAOs, VBOs, and Framebuffers with safe cleanup mechanisms.

Advanced GPGPU

Support for ping-pong buffer cycling, offscreen rendering, blitting, and SSBO simulation via float textures.

Integrated Math & Events

Built-in matrix mathematics (Perspective, LookAt) and seamless pointer, touch, and keyboard event routing.

GPU Profiling

Access precise hardware timing using disjoint timer queries to optimize your draw calls down to the microsecond.

Quick Start

Drop GiosGL.js into your project and set up a high-performance render loop in seconds.

<canvas id="glCanvas"></canvas>
<script src="GiosGL.js"></script>
<script>
    const canvas = document.getElementById('glCanvas');
    const engine = new GiosGL(canvas);

    // 1. Create Shaders
    const vs = `#version 300 es
        in vec2 position;
        void main() { gl_Position = vec4(position, 0.0, 1.0); }`;
    
    const fs = `#version 300 es
        precision highp float;
        out vec4 color;
        void main() { color = vec4(0.0, 0.8, 1.0, 1.0); }`;
    
    engine.createProgram('main', vs, fs, { position: 0 });

    // 2. Create Geometry
    const quad = new Float32Array([-1, -1, 1, -1, -1, 1, 1, 1]);
    const vbo = engine.createVBO(quad);
    const vao = engine.createVAO([{vbo: vbo, size: 2, type: engine.gl.FLOAT, location: 0}]);

    // 3. Render Loop
    engine.run((time, delta) => {
        engine.useProgram('main');
        engine.bindVAO(vao);
        engine.drawTriangles(4);
    });
</script>

Core API Documentation

Initialization & Context

new GiosGL(canvas)

Initializes the engine, applying performance hints and setting the internal resolution to 3840x2160.

resize(width, height)

Resizes the drawing buffer independent of CSS layout. Emits events to registered onResize listeners.

Shader System

createProgram(id, vsSrc, fsSrc, attribLocations?)

Compiles and links a GLSL ES 3.0 program. Automatically caches all active uniforms and attributes for ultra-fast lookup.

useProgram(id)

Binds a compiled program for drawing.

setUniform[1f|2f|3f|4f|1i|Matrix4fv](name, value)

Updates a uniform value on the currently bound program.

Buffers & Geometry

Method Description
createVBO(data, id?, usage?) Uploads geometry data to GPU RAM (STATIC_DRAW by default).
createIBO(data, id?, usage?) Uploads index data for element drawing.
createVAO(attribs, ibo?, id?) Maps buffer data to shader attributes securely.

Textures & Framebuffers

createTexture2D(id, width, height, options)

Creates a 2D texture. Options include pixel data, internal format, and min/mag filters (Defaults to NEAREST for sharp 4K).

createFramebuffer(id, options)

Generates an FBO attached to a texture. Essential for post-processing and GPGPU workflows.

createCubemap(id, urls, onLoad)

Asynchronously loads six images to form a skybox or environment map.

Utilities

Method Description
on(eventType, handler) Binds high-performance listeners for mouse, touch, and keyboard.
pickPixel(x, y) Reads back a specific pixel color array. Perfect for object picking.
saveScreenshot(filename?) Downloads the current framebuffer state as a PNG.
beginTimerQuery(name) Starts precise hardware performance profiling.
